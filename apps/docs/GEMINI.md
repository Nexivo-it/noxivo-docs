# Noxivo Docs - GEMINI Context

This project is a modern documentation website built using **Docusaurus v3**. It serves as the developer documentation for the **Noxivo Engine**, a high-performance, headless workflow engine for automating WhatsApp interactions.

## Project Overview

- **Purpose**: Documentation for Noxivo Engine (WhatsApp automation, Agentic Intelligence, Visual DAG Builder).
- **Architecture**: Docusaurus static site generator.
- **Main Technologies**:
  - **Framework**: [Docusaurus 3.10.0](https://docusaurus.io/)
  - **UI Library**: React 19
  - **Language**: TypeScript
  - **Content**: Markdown/MDX
  - **Styling**: Vanilla CSS (customized in `src/css/custom.css`)

## Building and Running

The project uses `npm` (or `yarn`/`pnpm`) for package management.

- **Install Dependencies**:
  ```bash
  npm install
  ```
- **Local Development**:
  ```bash
  npm run start
  ```
  Starts a local development server with hot-reloading at `http://localhost:3000`.
- **Production Build**:
  ```bash
  npm run build
  ```
  Generates static content into the `build/` directory.
- **Serve Production Build Locally**:
  ```bash
  npm run serve
  ```
- **Deployment**:
  ```bash
  npm run deploy
  ```
  Deploys the site (configured for GitHub Pages in `docusaurus.config.ts`).
- **Type Checking**:
  ```bash
  npm run typecheck
  ```
  Runs TypeScript compiler checks.

## Development Conventions

- **Documentation Location**: All documentation files are located in the `docs/` directory.
- **Routing**: The documentation is served at the root (`/`) of the site. The main landing page is `docs/intro.md` (via `slug: /`).
- **Sidebar**: The sidebar is autogenerated based on the filesystem structure of the `docs/` directory (defined in `sidebars.ts`).
- **Components**: Custom React components and pages are located in `src/components/` and `src/pages/`.
- **Assets**: Static assets like images and icons are stored in the `static/` directory.
- **Syntax Highlighting**: Prism is configured for code blocks, with additional support for `bash` and `json`.
- **Dark Mode**: Enabled by default and respects system color scheme preferences.
- **Plugins**: Uses `@docusaurus/remark-plugin-npm2yarn` to automatically generate Yarn/pnpm tabs for npm code blocks.

## Key Files

- `docusaurus.config.ts`: Main configuration for the Docusaurus site (title, URL, presets, theme config).
- `sidebars.ts`: Configuration for the documentation sidebars.
- `package.json`: Project dependencies and scripts.
- `docs/`: Markdown/MDX documentation source files.
- `src/css/custom.css`: Global CSS overrides and theme variables.
